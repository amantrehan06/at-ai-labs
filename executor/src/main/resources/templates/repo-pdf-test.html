üìö Available Endpoints

    Service health check and endpoint information
  • GET /api/v1/repo-pdf/info
    Get information about the current repository
  • GET /api/v1/repo-pdf/generate
    Generate PDF from current repository (downloads automatically)
  • GET /api/v1/repo-pdf/generate/{repoName}
    Generate PDF from specific repository by name
  • POST /api/v1/repo-pdf/generate-custom
    Generate PDF from custom repository path (JSON body with repoPath)

üí° How It Works

1. File Scanning: Recursively scans the repository for .java files
2. Content Extraction: Extracts package names, class names, and file content
3. PDF Generation: Creates a structured PDF with code analysis
4. Download: Automatically downloads the generated PDF file

Supported: Java classes, interfaces, enums
Ignored: target/, build/, .git/, .idea/, and other build directories
Content Limit: First 100 lines per file to keep PDF manageable
